# Signal Index Spec

Approved direction for the SkillJury public-site redesign.

## Core Thesis

SkillJury should feel like editorial infrastructure for AI-skill discovery:

- minimal in structure, not empty in content
- premium through typography, hierarchy, and restraint
- browse-first and compare-first
- anti-slop by making evidence explicit and inspectable

This is not a cinematic launch site. It is not a generic YC clone either. The site should borrow YC's clarity, literal IA, and proof-led density while keeping a stronger SkillJury signature through contrast, polish, and carefully limited motion.

## Visual System

- Palette: graphite shell, warm ivory content planes, one orange accent for emphasis only
- Typography: editorial serif for high-level page headlines; neutral sans for body, navigation, metadata, and controls
- Surfaces: borders and contrast do the work; no glassmorphism blanket, no multicolor fog
- Motion: shared-element continuity, subtle list reflow, one-time section reveals; no decorative hover choreography

## Product Rules

- One headline per screen
- One dominant action per zone
- Front-load real decision signals
- Rows beat cards for long result sets
- Trust signals stay separated by source
- Unknowns remain visible instead of being polished away

## Homepage

- Hero must explain the product in one pass
- Search is the primary action
- Discovery sections should compress into a tighter index instead of multiple marketing rails
- Leaderboard becomes a sharper "top skills now" surface, not a giant interaction playground
- FAQ should not dominate the landing surface

## Search / Browse

- Search and filters should read as one rail, not competing panels
- Results should prioritize comparability: name, source, category or compatibility, review signal, install signal, freshness
- Equal-weight glossy cards should be reduced or replaced with denser rows
- Metadata slots should stay consistent across items

## Skill Detail

- Top of page should answer: what is this, is it any good, should I install it
- Evidence lanes must stay distinct:
  - SkillJury verdict
  - source metadata
  - repository / external proof
  - community review signal
- Install command and compatibility need immediate visibility
- Supporting narrative comes after proof

## Anti-Slop Guardrails

- No fake certainty
- No decorative gradients that obscure information density
- No oversized marketing copy that hides product function
- No meaningless badge spam
- No motion that does not improve orientation
- No trust theater without inspectable evidence

## File Priorities

1. Shared shell and homepage
2. Search and listing surfaces
3. Skill detail and review summary hierarchy
